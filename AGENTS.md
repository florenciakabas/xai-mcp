# AGENTS.md — Source of Truth for AI Agents in This Repository

## What This Project Is

**xai-toolkit** is a Python package that provides ML model explainability as
plain-English narratives backed by quantitative evidence. It is exposed as an
MCP (Model Context Protocol) server that LLMs in VS Code Copilot agent mode
can call directly.

The user asks a question like "Why did the model flag this well as high risk?"
and gets back a deterministic English explanation powered by SHAP analysis — no
plots to decipher, no code to run.

## Architecture — The Golden Rule

**The LLM is the presenter, not the analyst.**

All analysis is done by pure Python functions. The LLM's only job is:
1. Understand the user's question
2. Choose the right MCP tool
3. Present the pre-computed result conversationally

This makes outputs **reproducible**: same question + same data = same answer.

## Project Structure

```
xai-toolkit/
├── src/xai_toolkit/         # All source code
│   ├── explainers.py        # Pure SHAP/importance computation (no MCP imports)
│   ├── narrators.py         # Structured data → English narratives (no LLM)
│   ├── plots.py             # Matplotlib → base64 PNG (supporting evidence)
│   ├── schemas.py           # Pydantic models — single source of truth for contracts
│   ├── registry.py          # ModelRegistry: load/list/manage models
│   └── server.py            # MCP server — THIN adapter, calls pure functions
├── tests/                   # pytest tests, mirrors src/ structure
├── models/                  # Serialized toy models (joblib)
├── data/                    # Toy datasets
├── docs/decisions/          # Architecture Decision Records (ADRs)
├── specs/                   # Tool specifications (written before implementation)
├── scenarios/               # YAML acceptance criteria (written before implementation)
└── .vscode/mcp.json         # MCP server config for VS Code Copilot
```

## Key Design Decisions

1. **Separation of concerns (ADR-001):** Pure Python functions have zero MCP
   dependency. The MCP server is a thin adapter. This means functions are testable
   with pytest alone and reusable in notebooks, REST APIs, or scripts.

2. **Deterministic narratives (ADR-002):** English output is generated by Python
   string formatting, NOT by an LLM. This ensures reproducibility and auditability.

3. **Pydantic schemas (ADR-004):** All tool inputs and outputs are defined as
   Pydantic models. These are the single source of truth for the contract between
   the MCP server and the pure functions.

## How to Build and Run

```bash
uv sync                          # Install all dependencies
uv run pytest                    # Run tests
uv run python -m xai_toolkit.server  # Start MCP server (stdio)
```

## Coding Standards

- **Python 3.11+**, type hints on all function signatures
- **Pydantic v2** for all data contracts
- **pytest** for all tests, prefer parametrized tests
- **Docstrings** on all public functions (Google style)
- Pure functions preferred — minimize side effects
- No MCP imports outside of `server.py`
- Narratives must be deterministic — no randomness, no LLM calls

## Tool Output Contract

Every MCP tool returns a dict with this structure:
```python
{
    "narrative": str,      # Plain English interpretation (REQUIRED)
    "evidence": dict,      # Structured numeric data backing the narrative
    "metadata": dict,      # model_id, timestamp, tool_version, data_hash
    "plot_base64": str,    # Optional base64 PNG for supporting visualization
}
```

## What NOT to Do

- Do NOT put computation logic in `server.py` — it's an adapter only
- Do NOT use LLM calls to generate narratives — they must be deterministic
- Do NOT import MCP or FastMCP in any file except `server.py`
- Do NOT modify tool output structure without updating `schemas.py` first
- Do NOT skip writing a scenario before implementing a feature
